Part Three Create a custom component

In this section we'll explore extending USWDS by creating a custom testimonial component not offered by the design system.

Create a the markup for a new component.

Visit _includes/testimonial.html. This will be where we will add the markup for our new component. Once the markup is filled out, it will appear beneath our custom card component!

Start by adding the basic skeleton of the component. We'll need an outer div, a body, and footer section. We'll call this component my-testimonial.

description _includes/testimonial.html

<div class="my-testimonial">
  <div class="my-testimonial__body"></div>
  <div class="my-testimonial__footer"></div>
</div>

Now lets add a quote to our testimonial body.

description _includes/testimonial.html

<div class="my-testimonial">
  <div class="my-testimonial__body">
    <p>
      “One of the best USWDS tutorials I’ve followed all the way to the end.”
    </p>
  </div>
  <div class="my-testimonial__footer"></div>
</div>

Now lets create a profile section which will feature a profile picture, name, and title.

description _includes/testimonial.html

<div class="my-testimonial">
  ...
  <div class="my-testimonial__footer">
    <img src="https://doodleipsum.com/100x100/avatar-5?bg=a5a8eb&shape=circle&n=1" class="my-testimonial__icon" alt="Profile pic">

</div>

Lastly, lets add a quotation icon for some visual flare! Visit the Icon component page, search for quote, and click the search result to copy the icons markup. Paste this markup directly after the opening my-testimonial div.

<div class="my-testimonial__body">

If you've recently done Part One of this tutorial, you probably remember why: we need to update the icon image path to point to its location in this project. We'll need to change the path we got when we copied the icon image from the USWDS website (/assets/img/sprite.svg) to the proper path in our project (/assets/uswds/img/sprite.svg).

  <div class="my-testimonial__body">

Now we've got some markup, but it doesn't really have any styling. Let's change that.

Add stylesheet partial.

Create a new stylesheet partial for our new component by entering the following in the free temrinal window: 

touch assets/css/components/_my-testimonial.scss

And then we'll import this new stylesheet into our USWDS custom styles.

description _theme/_uswds-theme-custom-styles.scss

@use "uswds-core" as *;
@forward "../assets/css/styles.scss";
@forward "../assets/css/_my-card.scss"; 

@forward "../assets/css/_my-testimonial.scss";

Start styling the component.

Just like using the color() function to use USWDS color tokens, we can use units() to allow native CSS properties to make use of USWDS spacing tokens. To get our base styles for our new component, add the following styles to the new stylesheet.

description assets/css/components/my-testimonial.scss

  @use "uswds-core" as *;
  
  .my-testimonial {
    background-color: color("indigo-10");
    padding: units(3);
  }

  .my-testimonial__icon {
    fill: color("indigo-30");
    stroke: color("indigo-30");
  }
  
  .my-testimonial__body p {
    text-align: center;
  }
  
  .my-testimonial__footer {
    display: flex;
    flex-direction: row;
    margin-top: units(3);
  }
  
  .my-testimonial img {
    @include u-circle(10);
  }
  
  .my-testimonial__profile {
    margin-left: units(3)
  }
  
  .my-testimonial__profile-name {
    color: color("indigo-70v");
  }
  
  .my-testimonial__profile-title {
    margin-top: units(0);
  }

Can you spot all the uses of USWDS tokens?

Styling the testimonial icon

USWDS uses variables to maintain consistent styles across different definitions. Let's create a variable for our icon stroke and fill at the top of our stylesheet.

description assets/css/components/my-testimonial.scss

@use "uswds-core" as *;


$my-testimonial-icon-color: color("indigo-30");


...

.my-testimonial__icon {

  fill: color("indigo-30");
  stroke: color("indigo-30");

  fill: $my-testimonial-icon-color;
  stroke: $my-testimonial-icon-color;

}

Using USWDS utility mixins.

USWDS also offers an extensive library of utility mixins which you can use to simplify style setting. Let's use some of the typeset mixins to customize our font weight and size. It's standard convention to include your mixins at the top of each style definition.

description assets/css/components/my-testimonial.scss

.my-testimonial__body p {

  font-size: size("heading", "xl");
  font-weight: font-weight("bold");
  text-align: center;
}

...

.my-testimonial__profile-title {
  margin-top: units(0);

  font-size: font-size("body", 5)
}

Using uswds theme tokens with mixins

Finally, lets use a combination of utility mixins and uswds theme settings to display how you can add uniformity across your project! In this example, we'll add the border settings from step one to help our new component sit nicely with the original card.

description assets/css/components/my-testimonial.scss

.my-testimonial {
  @include u-margin-x(1);
  @include u-border($theme-card-border-width);
  @include u-radius($theme-card-border-radius);
  background-color: color("indigo-10");
  padding: units(3);
}

And voila! You've used theme settings, custom styles, USWDS tokens, and utility mixins to theme, customize, and extend USWDS into creating our very own component! Great work!
